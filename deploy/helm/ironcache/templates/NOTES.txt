{{- if .Values.cluster.enabled }}

Raft cluster mode is ON ({{ int .Values.replicas }} nodes, 16384 slots split evenly).

IMPORTANT -- scaling: do NOT resize this cluster with `kubectl scale`, HPA, or a bare
`helm upgrade --set replicas=N`. Slot ownership is static at bootstrap and membership is
a Raft quorum, so a blunt replica change orphans slots or splits the quorum. Read the
runbook (drain/decommission + reshard procedures) before resizing: deploy/SCALING.md.

{{- if not .Values.clusterSecret.value }}

WARNING: clusterSecret.value is empty -- a RANDOM secret was generated on install.
It is now PRESERVED across `helm upgrade` (the chart looks up the existing Secret), so
a bare upgrade no longer splits the cluster. BUT under GitOps (Argo/Flux) or any
`helm template`/`--dry-run` pipeline the lookup is blind and the value REGENERATES every
sync -- so for GitOps and reproducible/production installs you MUST set an explicit stable
clusterSecret.value (or clusterSecret.existingSecret). Generate one: `openssl rand -hex 20`.
