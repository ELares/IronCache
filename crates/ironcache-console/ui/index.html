Failover

                admin, destructive
              

              

                Promote THIS node to primary for the slots it replicates. The engine
                refuses it unless this node is an in-sync replica, so it cannot lose
                committed writes; FORCE / TAKEOVER are not offered. Type FAILOVER to
                confirm.
              

              

                Admin token required to trigger a failover.
              

              

                
                
                
              

              

            


            

              

                
Add node

                admin
              

              

                CLUSTER MEET: introduce a node at host:port to the cluster. The engine
                validates the address and the leader commits the membership.
              

              
Admin token required to add a node.

              

                
                
                
                
                
              

              

            


            

              

                
Remove node

                admin, destructive
              

              

                CLUSTER FORGET: remove a node from the cluster view. Enter the EXACT
                node id to remove it (the engine refuses forgetting this node itself or
                an unknown id).
              

              
Admin token required to remove a node.

              

                
                
                
              

              

            


            

              

                
Migrate slot

                admin, destructive
              

              

                CLUSTER SETSLOT: drive an online slot migration. MIGRATING / IMPORTING
                tag the source / destination, NODE commits the FLIP (final ownership),
                and STABLE clears the state. Enter the slot to confirm.
              

              
Admin token required to migrate a slot.
